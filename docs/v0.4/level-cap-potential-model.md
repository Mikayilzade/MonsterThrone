# Поддерживаемый потолок уровня и постоянный потенциал v0.4

Статус: **[AI PROPOSAL]**, основанный на **[USER DIRECTION]**: если характеристики позволяют развитие, скорость `×1` равна одному уровню за 60 секунд мирового времени; победа над существенно более сильной целью даёт процентное ускорение.

Подробная проверка вариантов находится в [`level-cap-potential-experiment-results.md`](level-cap-potential-experiment-results.md).

# 1. Главный принцип

Нужно разделить три вещи:

```text
постоянный потенциал
→ определяет, до какого уровня существо вообще способно дорасти

освоенная часть потенциала
→ определяет текущий уровень и постоянные выраженные характеристики

временная сила
→ действует сразу, но не открывает постоянный уровень
```

Уровень не создаёт бесплатные характеристики. Он постепенно осваивает уже полученный потенциал.

# 2. Три набора характеристик

## 2.1. Общий постоянный потенциал

`totalPotentialStats` включает устойчивые возможности души и тела:

- наследованный потенциал;
- поглощённые постоянные характеристики;
- закреплённые мутации;
- устойчивые органы;
- результат длительной адаптации;
- редкие постоянные магические изменения.

Он состоит из:

```text
totalPotentialStats
= expressedPermanentStats
+ latentPotentialStats
```

## 2.2. Выраженные постоянные характеристики

`expressedPermanentStats` — часть потенциала, которую текущее тело уже освоило и способно постоянно использовать.

Из них рассчитывается текущий уровень.

## 2.3. Временные характеристики

`temporaryStats` добавляются после расчёта постоянной силы.

Источники:

- свежее мясо;
- зелья;
- заклинания;
- артефакты;
- ярость;
- временное перенапряжение органа.

Они:

- не повышают `supportedLevelCap`;
- не восстанавливают душу через правило `+100`;
- не становятся постоянными автоматически;
- могут позволить совершить событие, которое позже даст постоянный потенциал или импульс развития.

# 3. Расчёт поддерживаемого потолка

Каждый план тела задаёт:

```text
supportWeights[]
statCostPerLevel[]
bodyEfficiency
supportChannels[]
```

Для обычного четырёххарактеристического тела используются сила, выносливость, ловкость и интеллект.

Сначала каждая характеристика переводится в собственную поддержку уровня:

```text
support_i = totalPotentialStat_i / statCostPerLevel_i
```

Начальный ориентир стоимости:

```text
10 постоянных очков подходящей характеристики
≈ 1 единица поддержки уровня
```

Это не означает, что любые 10 очков всегда дают ровно один уровень: итог зависит от всего плана тела.

## [AI PROPOSAL] Мягкое правило узкого места

Для первого стенда:

```text
supportedLevelCap
= floor(
    bodyEfficiency
    × (Σ weight_i × support_i^0.25)^4
  )
```

Условия:

- сумма весов равна `1`;
- обычный жизнеспособный план тела не отдаёт одной основной характеристике более `45%` общей поддержки;
- критические системы сохраняют ненулевой вес;
- аморфные, коллективные, неживые и чисто магические сущности могут использовать другой набор каналов вместо человеческих четырёх характеристик.

Эта формула является взвешенным степенным средним с мягким штрафом за провал одной из важных систем.

# 4. Почему не простое среднее

Простое арифметическое среднее позволяет существу вложить почти всё в одну самую выгодную характеристику и заменить ею отсутствующие жизненные системы.

Чистый минимум делает любую слабую характеристику абсолютным потолком и уничтожает специализацию.

Чистое геометрическое среднее слишком сильно наказывает даже разумные перекосы.

Степенное среднее с показателем `0.25`:

- награждает профиль, соответствующий плану тела;
- допускает сильную специализацию;
- не позволяет одной огромной характеристике полноценно заменить остальные;
- остаётся однородным: уменьшение всех выраженных характеристик на `10%` уменьшает текущий уровень ровно на `10%`.

# 5. Текущий уровень

Текущий уровень рассчитывается той же функцией, но от уже выраженных характеристик:

```text
currentLevel
= LevelSupportFunction(expressedPermanentStats)
```

Потолок:

```text
supportedLevelCap
= LevelSupportFunction(totalPotentialStats)
```

Разница между ними — ещё не освоенный запас:

```text
levelRunway = supportedLevelCap - currentLevel
```

# 6. Созревание уровня

При нормальных условиях:

```text
×1 = +1 уровень за 60 секунд мирового времени
```

Каждая завершённая минута созревания не создаёт новые характеристики, а переводит часть `latentPotentialStats` в `expressedPermanentStats` так, чтобы функция текущего уровня выросла примерно на единицу.

Перенос идёт по структуре скрытого потенциала. Если существо накопило преимущественно силу, следующие уровни прежде всего выражают силу; остальные уже выраженные характеристики не уменьшаются.

Рост останавливается, если:

- `currentLevel >= supportedLevelCap`;
- критический голод не позволяет строить и поддерживать тело;
- тело разрушено сильнее допустимого;
- подходящие органы не сформированы;
- энергетическая зона подавляет развитие;
- идёт тяжёлая перестройка, болезнь или метаморфоза;
- душа или особый эффект прямо блокирует созревание.

# 7. Победный импульс

Рабочий ориентир из отдельного эксперимента:

```text
+1% скорости за один эффективный уровень разрыва
максимум +300%
действует сильнейший актуальный импульс
```

Эффективный разрыв учитывает:

- личный вклад;
- реальный риск;
- помощь группы;
- временную силу;
- повторяемость безопасной победы;
- качество понимания произошедшего.

Импульс:

- ускоряет только выражение уже доступного потенциала;
- не создаёт новый потолок сам по себе;
- угасает по мере приближения к уровню побеждённой цели;
- не действует в диапазоне `900+`;
- не складывается полностью с множеством слабых побед.

Поглощение тела цели может отдельно добавить скрытый постоянный потенциал.

# 8. Получение постоянного потенциала

## 8.1. Созревание молодняка

Потомок имеет наследованную схему взрослого тела, но не получает её мгновенно.

`maturationReserve` постепенно превращается в постоянный потенциал по видовому графику взросления. После завершения взросления возраст сам по себе не создаёт бесконечный рост.

## 8.2. Поглощение

Для первого стенда предлагается сохранить знакомый масштаб:

- одна ценная свежая порция усваивает примерно `0.5–1%` подходящей постоянной характеристики донора;
- вся свежая туша содержит суммарно около `3–5%` извлекаемого постоянного потенциала;
- при идеальной свежести, совместимости, органах и обработке максимум может доходить примерно до `7%`;
- добыча делится между едоками;
- повторное питание одной и той же линией даёт уменьшающуюся отдачу;
- несовместимость может уменьшить прирост, направить его в мутацию или причинить вред.

Постоянное поглощение в первую очередь увеличивает `latentPotentialStats`. Немедленный чудовищный скачок обеспечивает отдельный временный эффект свежего мяса.

## 8.3. Значимая адаптация

Обычное повторение безопасного действия не создаёт бесконечные характеристики.

Постоянная адаптация требует одновременно:

- реальной нагрузки;
- восстановления;
- подходящего питания или энергии;
- отклика тела;
- разнообразия условий;
- отсутствия полной насыщенности этой адаптации.

Стартовый технический ориентир:

```text
обычная значимая адаптация:
около 1–3 уровней постоянного потенциала в час
```

Это общий эквивалент, распределяемый между затронутыми характеристиками, а не бесплатные уровни текущего развития.

## 8.4. Органы и мутации

Орган или стабильная мутация может:

- добавить постоянные характеристики;
- изменить `supportWeights`;
- изменить `statCostPerLevel`;
- повысить `bodyEfficiency`;
- создать новый канал поддержки;
- обменять универсальность на специализацию.

Ориентир для первого стенда:

- малая стабильная перестройка: `+1–5` уровней эквивалентного потолка;
- заметная мутация: `+5–20`;
- крупная редкая перестройка: `+20–50`;
- более сильные изменения требуют крупной цены, события или уникальности.

Эти числа не являются гарантированной чистой прибавкой: мутация может одновременно поднять одну ветвь и ухудшить другую.

# 9. Смерть

## [AI PROPOSAL] Обычная смерть

При потере около `10%` уровня:

```text
expressedPermanentStats ×= 0.9
утраченная выраженная часть → latentPotentialStats
```

Стабильный общий потенциал обычно сохраняется в душе. Поэтому существо:

- становится примерно на `10%` слабее;
- перерождается в кольце нового уровня;
- способно заново освоить прежний потенциал со скоростью созревания;
- всё равно не восстанавливает душу простым возвращением старых уровней, потому что для полного восстановления требуется новый личный прогресс `+100` от прежней точки.

Отдельно могут быть потеряны:

- нестабильные мутации;
- не закреплённые органы;
- временные характеристики;
- часть повреждённого прямым ударом потенциала;
- особые свойства при пожирании ядра или разрушении души.

# 10. Потолок ниже текущего уровня

Редко существо может потерять важный орган или устойчивый потенциал так, что:

```text
currentLevel > supportedLevelCap
```

Тогда возникает `unsupportedLevelDebt`:

- дальнейший рост невозможен;
- эксплуатационная стоимость тела повышена;
- часть органов ограничивается или засыпает;
- организм пытается восстановить поддержку;
- при долгом провале уровень постепенно оседает к поддерживаемому значению;
- мгновенного обвала без особой причины нет.

# 12. Что остаётся экспериментальным

- показатель степенного среднего `0.25`;
- точные веса планов тела;
- стоимость одной единицы уровня по характеристикам;
- скорость видового взросления;
- коэффициенты поглощения;
- насыщение повторной пищей;
- темп адаптации;
- размеры мутационных прибавок;
- точное поведение `unsupportedLevelDebt`;
- доля нестабильного потенциала, теряемая при смерти.

Эти значения должны быть видимы в инспекторе и настраиваться без переписывания ядра симуляции.
